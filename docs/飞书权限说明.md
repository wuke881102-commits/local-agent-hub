# 飞书应用权限说明（最小权限原则）

> 下列权限用于「飞书文档管理」能力；纯本地的「本地文件内容生成」「自动化提炼」不需要任何飞书权限。

本工具严格遵循**最小权限原则（least privilege）**：

- 登录时只请求下面这一份**最小 scope 集**，而不是 `lark-cli --recommend` 拉来的一大堆写 / 删 / 分享 / 邮箱权限；你在飞书同意页只会看到这一最小集。
- 绝大多数是**只读**权限；少数「写」权限**只有在你于产品内点「确认」后才会真正调用**（写回文档、发群消息、建任务）。
- 以**你本人的身份**授权（设备码登录），只能访问你原本就有权限看到的内容。
- 该清单由代码里实际的飞书调用面审计得出（`backend/app/feishu/cli.py` 的 `_MIN_LOGIN_SCOPES_LIST`），并用 `lark-cli auth check --scope "…"` 逐个验证为合法且够用。

## 只读权限

### 文档 / 云盘

| Scope | 用途 |
|---|---|
| `docx:document:readonly` | 读文档正文（docx） |
| `docs:document.content:read` | 读文档正文 |
| `docs:document.media:download` | 下载文档内图片 / 媒体 |
| `drive:drive.metadata:readonly` | 读云盘文件元信息 |
| `space:document:retrieve` | 枚举云盘文件 / 多维表格（**建索引必需**） |
| `search:docs:read` | 云盘搜索，含「共享给我的」（**建索引必需**） |
| `drive:file:download` | 下载云盘文件（PDF 等） |

### 知识库 / 电子表格 / 多维表格 / 幻灯片

| Scope | 用途 |
|---|---|
| `wiki:space:read` · `wiki:node:read` | 读知识库空间 / 节点 |
| `wiki:space:retrieve` · `wiki:node:retrieve` | 枚举知识库空间 / 节点（**建索引必需**） |
| `sheets:spreadsheet:read` · `sheets:spreadsheet.meta:read` | 读电子表格及元信息 |
| `base:app:read` · `base:table:read` · `base:record:read` | 读多维表格（应用 / 数据表 / 记录） |
| `slides:presentation:read` | 读幻灯片 |

### 日历 / 会议纪要 / 通讯录 / 群

| Scope | 用途 |
|---|---|
| `calendar:calendar:read` · `calendar:calendar.event:read` | 读日历 / 日程 |
| `vc:record:readonly` | 读会议录制 |
| `minutes:minutes.search:read` | 搜索妙记（会议纪要提炼） |
| `contact:user.base:readonly` · `contact:user:search` | 读用户基本信息 / 搜索用户（组织架构图） |
| `im:chat:read` | 读群列表（协作分发选目标群） |

## 写权限 —— 均需在产品内「确认」后才触发

| Scope | 用途 | 触发时机 |
|---|---|---|
| `docx:document:create` | 写回：创建文档 | 你在「写回确认」弹窗点确认后 |
| `im:message` · `im:message.send_as_user` | 协作分发：以你的身份发群消息 | 你点「直接发送」后 |
| `task:task:write` | 创建任务（会议待办回写） | 你确认回写待办后 |

> 说明：`lark-cli auth check --dry-run` 只校验到 `im:message` 就放行，但真正发送时飞书强制要求更细的 `im:message.send_as_user`，故两者都请求，确保新授权用户可用。

## 会话

| Scope | 用途 |
|---|---|
| `offline_access` | 刷新令牌，保持登录（免反复授权） |

## 如何配置你自己的飞书应用

1. 到[飞书开放平台](https://open.feishu.cn/)创建一个**企业自建应用**。
2. 在「权限管理」里添加上面列出的 scope。**只授予你需要的能力**即可——例如不打算发消息 / 写回，就不必勾选对应的写权限，只读功能不受影响。
3. 把应用的 `App ID` / `App Secret` 填入 `backend/.env`（`FEISHU_APP_ID` / `FEISHU_APP_SECRET`），或按 `lark-cli` 的交互流程配置。
4. 首次登录时，本工具会以 `lark-cli auth login --scope "<上面这份最小集>"` 发起授权——你在同意页看到的就是这一最小集。

> 若后续新增了需要更多权限的功能，请同步更新 `backend/app/feishu/cli.py` 的 `_MIN_LOGIN_SCOPES_LIST`，并用 `lark-cli auth check --scope "…"` 复核确切的 scope ID。
